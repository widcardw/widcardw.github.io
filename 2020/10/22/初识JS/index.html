随机数生成器
随机范围
剩余数量：∞
 
 

这算是我写的第一个比较满意的前端程序了吧

我想随机数生成器大家应该都会用，所以使用说明也不用我来说了吧。

之前写的那个版本非常的难看，毕竟也只是稍微用 html 写了写，界面不怎么行，也没有文档说明。在这里的话我就稍微完善以下这个网页吧。

想看代码的话可以直接按 F12，在这里我也就不想放详细的代码了。

基本逻辑：首先，当数据范围比较小的时候，生成一个枚举型的 Array，即数组的数值等于它的索引加一。接着，通过随机数抽取索引对应的数，并将该索引的元素移出数组，数组大小自减。最后数组长度为 0 时，停止抽取。

但是，这也带来一个问题，就是当数据范围很大的时候，例如范围是 1 亿，那么我们就得先生成一个含有 1 亿个元素的枚举型数组。这样会给浏览器造成很大的负担，甚至会直接闪退。所以，我打算在数据量较大时，直接生成指定范围内的随机数；如果涉及到不重复的抽取，那就采用之前的方法，即将已经抽过的数放在一个黑名单数组中，如果抽到的数在黑名单里，那么就会重新抽取，直到抽到不在黑名单的数为止。

观众：那么，万一到后面抽完的时候岂不是要循环很多次才能抽到不同的数吗？那这个怎么解决呢？

答：你能徒手点完一千一万次，我真的佩服你。

所以，数据范围大于一万的时候，我就只允许你抽取数据范围的十分之一，或者只允许你抽取一万次。我相信不会有人真的拿个连点器专门用来测我这垃圾玩意。

这是我认为相对来说比较方便的算法了，如果有大佬能够提供更好的想法，欢迎留言。

注意事项：当重新输入范围，勾选或取消勾选“不重复”都会重置数组，也就是说，会从给定的范围内重新抽取。这是我个人觉得应该加入的一些想法，不知道大家怎么想。

另外，为了写这玩意我还特意临时去学了 CSS，发现这玩意真的是不写不知道，一写吓一跳。在有些方面，这玩意挺方便的，主要是用于排版，但是不知道后续会不会遇到比较麻烦的问题（我觉得可能主要是浏览器和手机端的适配，到时候有 BUG 了我再改，比如把范围调大之后抽取的数就会上下反复横跳，因为我加了 flex-wrap，为了适应屏幕宽度而换行，结果效果还是很差，到时候如果有机会的话再改吧）。

对了，之前一个自定义字体大小被我删了，很抱歉，手机端的话可能就没法抽取很大的数了，因为会显示不下。（这就是 CSS 吗？i 了 i 了）

那就这样了。
